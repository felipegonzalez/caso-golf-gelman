Problema

Queremos entender modelar la probabilidad de xito de putts de Golf (putts: tiros relativamente cerca del hoyo que buscan que la pelota ruede al hoyo o muy cerca de l), y cmo depende el xito de la distancia del tiro. Quisiramos inferir qu tan precisos son los profesionales en sus tiros.

Anlisis conceptual

Podemos pensar en cada intento que hace un golfista como una prueba independiente que puede resultar en xito o fracaso. La probabilidad de xito depende de la distancia, as que podramos usar regresin logstica con una variable de entrada (la distancia \(x\) al hoyo).

Espacio de observaciones

Usaremos datos de tiros de profesionales que incluyen la distancia al hoyo y si el tiro fue exitoso o no.

El espacio de observaciones que esperamos del tipo \((x, y)\) donde \(x\) es la distancia del putt y \(y\) indica si se logr o no. Probablemente tendremos los datos agregados: para cada distancia aproximada \(x\) tendremos un conteo de intentos y xitos sobre los tiros de los jugadores profesionales.

Estadsticas resumen

Consideraremos el porcentaje de xitos para cada distancia dada en las observaciones (cada cubeta de distancias) como resumen principal de inters.

sed -n '/^transformed/,/\}/p' ../stan/modelo.stan
transformed parameters {
  real<lower=0, upper=1> prob_exito[p];
  for(i in 1:p) {
   prob_exito[i] = inv_logit(beta_0 - beta * x[i]); 
  }

Desarrollo del modelo

Consideraremos que las observaciones del nmero de xitos \(e(x)\) a una distancia de \(x\) metros es

\[e(x) \sim \textrm{Binom}(n(x), p(x))\] y ponemos, segn regresin logstica:

\[\log \left (\frac{p(x)}{1 -p(x)} \right ) = \beta_0 - \beta x.\] Para completar el modelo es necesario poner distribuciones iniciales a los parmetros \(\beta\). Como \(\beta_0\) es la tasa de xito a distancias muy cortas, \(\beta_0\) debe ser relativamente grande. Ponemos por ejemplo

\(\beta_0 \sim N(6, 1),\)

de forma que a distancia cercana a cero, la probabilidad de xito est entre

inv_logit <- function(z) 1 / (1 + exp(-z))
c(inv_logit(4), inv_logit(8))
[1] 0.9820138 0.9996646

Para \(\beta\) es necesario tener alguna informacin adicional acerca del problema. Por ejemplo, para 5 metros esperamos que haya una cantidad considerable de fallas, pero tambin varios xitos. La probabilidad de xito a 500 metros (con \(\beta_0=6\))es

\[p(500) = \frac{1}{1 + e^{-6 - 500 \beta}}\] Si igualamos a 0.5 y despejamos, obtenemos \[ \beta = 0.012 \] Pondremos entonces \[\beta\sim N(0.015, 0.0025)\] que nos da un rango para la probabilidad de xito a 2 y 5 metros de

c(inv_logit(8 - 0.010*200), inv_logit(4 - 0.020*200))
[1] 0.9975274 0.5000000
c(inv_logit(8 - 0.010*500), inv_logit(4 - 0.020*500))
[1] 0.952574127 0.002472623

Verificacin posterior dentro de muestra

Ahora podemos hacer la verificacin ms bsica, que compara el ajustado con los valores reales. Esto nos indica problemas potenciales de desajuste que requieren reevaluar nuestro modelo. Estas verificaciones tambin se pueden hacer fuera de la muestra (predictivas), lo cual es ms apropiado en algunos contextos (por ejemplo, si el propsito principal es predecir).

Tenemos una seccin para generar tasas de xito simuladas (nuestro resumen principal) a partir del modelo ajustado:

sed -n '/^generated/,/\}/p' ../stan/modelo.stan
generated quantities {
  real prob_sim[p];
  for(i in 1:p){
    prob_sim[i] = binomial_rng(n[i], prob_exito[i]);
    prob_sim[i] = prob_sim[i] / n[i];
  }

El resultado es bastante malo. Tenemos sobreprediccin para tiros cortos y subpredicciones para tiros largos. En este punto, podramos agregar variables derivadas (por ejemplo splines o trminos polinomiales) para corregir este desajuste. Este camino es factible, pero muchas veces se puede resolver analizando particularidades de nuestro problema, e introduciendo variables interpretables que nos permitan aprender del fenmeno que estamos modelando.

Ideas para siguiente iteracin

  • Usar splines en la distancia para hacer ms flexible el ajuste
  • Regresar al anlisis conceptual y usar la geometra del problema
